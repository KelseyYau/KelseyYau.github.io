angular-resolve和路由守卫

发布 : 2019-04-02 分类 : 前端框架 浏览 :

参考angular中文网 路由守卫

参考angular router文章 angular router

1.介绍

之前在使用angular框架进行开发时，由于公司对于一些基础的组件和模块已经进行了封装，所以使用时，并没有过多的实践关注路由守卫的一些具体使用，只是停留在概念上，很快又忘记了。于是乎，记录下来，方便下次使用。

2. 情景模拟

在日常的开发中，可能总会有一些需求：

  • 不同的用户，导航的组件不同
  • 在进入某一路由初始化组件前，需要先请求一些数据
  • 在离开当前组件时，需要执行一些操作

上述需求，均对应有不同的路由守卫来执行相应的操作

三种主要路由守卫钩子：

  • Resolve
  • CanActivate
  • CanDeactivate

3.使用介绍

3.1 resolve

resolve钩子，是用于在路由跳转并在生命周期钩子ngOnit()之前，先获取用于初始化组件的数据。

如： AppComponent => DemoComponent

初始化DemoComponent组件之前，需要先异步请求获取到初始化组件所需的数据，我们通过依赖注入，注入DemoResolverService来进行操作，通过该方法可以返回一个 Promise、一个 Observable 来支持异步方式，或者直接返回一个值来支持同步方式。所需文件范例如下：

demo-resolve.service.ts

demo-resolve服务，用于处理resove方法，返回异步请求结果，需要注意的是resolve需要声明泛型，如本例声明的是一个any类型的可观察对象，具体可根据实际业务声明

demo-routing.module.ts

demo组件路由设置模块，注入demo-resolve.service 服务

demo.component.ts

demo组件

3.2 CanActivate

在web应用中，存在权限控制类的场景，根据访问者的情况决定是否授予访问某个组件的访问权限，CanActivate就是用于实现此类操作的钩子。

具体操作如下：

auth.guard.ts

如下代码，本质上AuthGuard是服务，通过依赖注入的方式，注入到路由文件中

app-routing.module.ts

路由配置文件，添加守卫

app.module.ts

模块文件要注入守卫的服务，即auth.guard.ts

通过设置，这样只有用户登录，并在sessionStorage中存储有用户登录信息后，CanActivate守卫返回true，才能跳转到对应的视图，否则返回登录视图

3.3 CanDeactivate

官方文档给出的解释是处理未保存的修改。常见应用场景，例如我们去注册一个至关重要的账号前填写各种调查户口似的资料。啪啪啪一顿猛操作，填完了所有表单，这时一般你有两个选择：（1）不提交保存，离开当前页。（2）提交，等待上传成功后跳转成功页面。这个时候，CanDeactivate钩子就充当了一个一品御前侍卫，替你把好关。
