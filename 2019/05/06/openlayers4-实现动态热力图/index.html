openlayers4-实现动态热力图

发布 : 2019-05-06 分类 : 前端地理信息可视化 浏览 :

参考Openlayers API ol.layer.Heatmap

参考heatmap.js heatmap.js

1.介绍

openlayers的热力图图层是依据开源库”heatmap.js”封装而成，保留了热力图的相关的配置如半径，模糊度，颜色等的设置等，具体设置见api，这里不再赘述。

2.实现

通过查阅api和官网示例，发现官网示例使用的kml格式文件去加载热力数据文件，而热力图层的数据源source是矢量数据源(ol.source.Vector)，所以加载其他矢量格式的热力数据如(GeoJSON)或者自己生成矢量要素(ol.Feature)构成数据源也是可行的。只是要注意的一点是热力图显示权重依据的是矢量要素要有”weight”属性。

如上图，weight属性可以是字符串或者函数，weight取值范围是0-1，值越大，权重越高。在本例中，我的weight属性是数字类型的发现也没问题呢，但是为了规范，还是转一下字符串吧。

本例实现动态变化的热力图是采用GeoJSON格式的数据，为了实现较为简单数据的动态，直接使用setInterval函数，实际使用中，应该是有时间选项，通过选择时间节点，显示该时间节点的热力图，业务逻辑上操作，可能会多种多样，这里就不一一展开，这里只实现。
